# 项目级 Codex 工作规则

## 项目概况

- 项目名称：coders-life
- 项目类型：单页 HTML 小游戏
- 技术栈：HTML、CSS、原生 JavaScript、本地 JSON/JS 数据文件
- 主要入口：`程序员生存模拟器_增强版.html`
- 主要资源：`程序员生存模拟器_弹窗库_2026-06-18.js`、`data/`、`docs/`、`tools/`

## 工作规则

- 保持当前视觉风格，不因功能迭代重做整体 UI。
- 主线保持“程序员生存模拟器”，弹窗内容服务游戏循环，不改成纯段子集合。
- 修改弹窗库时优先同步保留结构化来源文件，避免只改生成后的 JS。
- 涉及存档、随机事件、数值结算时，优先做浏览器内手动验证。

## 运行与验证

- 运行方式：直接双击 `程序员生存模拟器_增强版.html`，或用浏览器打开 `file:///E:/desktop/coders-life/程序员生存模拟器_增强版.html`。
- 基础验证：页面能打开，职业选择可进入游戏，行动按钮能生成事件，控制台无阻断错误。
- 弹窗库验证：浏览器控制台检查 `window.PROGRAMMER_POPUP_POOL.length` 应为当前弹窗池数量。

## 风险点

- 本项目是本地 file 页面，浏览器对本地文件、localStorage 和脚本加载可能有安全策略差异。
- `data/legacy-bad-encoding/` 是历史编码异常材料归档，默认不要作为正式数据源继续生成。
- 文件名包含中文，脚本处理时显式使用 UTF-8，避免乱码写回。
