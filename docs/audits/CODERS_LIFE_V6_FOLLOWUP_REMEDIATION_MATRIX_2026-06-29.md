## Batch Simulation Gate

`scripts/simulateBatch.ts` now executes real monthly multi-action plans through `planMonth` and records `averageMonthlyPlanSize`. The former `createReachabilityProbe` synthetic terminal-state check was removed; ending coverage is now reported from actual scenario outcomes as `coveredEndings`.

Default `npm run simulate:batch` is a local release smoke gate: all 8 strategies, `frontend`, `tier2`, 1 seed, 36 months. Larger offline sweeps remain available through `SEEDS_PER_SCENARIO`, `CAREERS`, `CITIES`, and `MAX_MONTHS`.

## Validation Commands

These commands must be recorded before creating the Draft PR:

```bash
npm ci
npm test
npm run build
npm run simulate
npm run simulate:batch
git diff --check
```

For local iterative work, `simulate:batch` may be run with constrained environment variables. The final PR description must distinguish quick local gates from any full offline batch simulation.
